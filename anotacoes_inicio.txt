NOVAS INSTRUÇÕES: 
	Hardware de interrupções: GETTIME (ms), LOCK, RELEASE, SETQUANTUM, GETQUANTUM (ciclos), UNBLOCKINST
	Memória de programa: GETPC, SETPC, STORE (mesma STORE da memória principal, ela vai tomar conta do barramento)
	No processador: SETHI, SETLO


Necessário para o processador e o SO:

- MMU (Unidade de gerenciamento de memória)
	Transcreve endereço virtual para endereço real e é responsável pela alocação das páginas em memória principal ou em disco
- Processador:
	- Sinalizador de conclusão de instrução
	- Criação de uma unidade de interrupção:
		- Essa unidade terá que ter como ser resetada
		- Executará um código (passando por cima da memória de programa) quando tiver que executar
		- Terá uma instrução de bloqueio de mutex para não escalona (LOCK e RELEASE)
		- Terá uma forma de serem carregadas instruções a ela uma unica vez
		- Tratará o momento de escalonar (fim de programa ou tempo quantum) de acordo com o sinalizador de conclusão de instrução
		- Contar as interrupções lançadas pelo relógio para contar o tempo "quantum"
			50MHz = 2*10^(-8) = 20ns => Se uma instrução for executada em média em 4 ciclos 80ns = 0.08us
			Contar a cada 10us? => Daria tempo de serem executadas 120 instruções
		- Ter como obter o tempo ligado em ms
	**INTRUÇÕES:
		- GETTIME
		- LOCK
		- RELEASE
		
	**PENSAMENTO DE FUNCIONAMENTO:
		- A unidade de interrução executaria um pequeno código de programa que substitui linha por linha da memória de programa do processador a partir de um registrador de kernel
		- O ato de escalonar seria essa unidade chamar o kernel de SO para escalonamento, esse executaria em memória de programa (mutável) alternado o valor desse registrador de kernel, finalizaria o processo (atômico???) lançando interrupção o que colocaria em execução a unidade de interrupção que utilizaria o valor desse registrador de kernel para substituir a memória de programa e colocar lá o próximo programa a executar, antes de seu fim, sempre, a unidade de interrupção substituiria o registrador de kernel por um registrador específico que é o endereço do escalonador de processos do SO.
	
- Estados dos processos:
	- Em execução
	- Pronto
	- Bloqueado
